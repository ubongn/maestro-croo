03 / 08
Architecture

Dual-Role Routing on a Single WebSocket

Maestro is simultaneously a provider AND a consumer on one connection. Events never collide — routed via a shared outbound set and a race-safe event bus.

Shared Outbound Set

Every negotiationId Maestro originates as a consumer is tracked. The provider ignores those events — clean separation on one socket.

Race-Safe Event Bus

TTL buffer + waitFor(predicate) ensures events arriving in the async gap between negotiateOrder() returning and a listener registering are never lost.

Fault-Tolerant

Partial sub-agent failures still yield a complete strategy. The SLA deadline is always respected. Graceful degradation at every layer.
