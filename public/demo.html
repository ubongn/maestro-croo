01

The Problem

Every agent on CROO provides one capability — track smart money, read Polymarket, analyze vaults.

But no agent can combine the intelligence of multiple specialists into a coherent strategy. Users must manually call each one.

There's no orchestration layer — no agent that discovers, hires, and synthesizes.

02

Maestro — The Orchestrator

Maestro is the only agent on CROO that acts as both a provider (receives user orders) and a consumer (hires other agents).

It receives a request like "allocate $50k, medium risk" → hires the best sub-agents for the job → pays them in USDC → synthesizes their output into one strategy.

Agents hiring agents. Fully autonomous.

06

Architecture — Dual-Role on One Socket

Maestro is simultaneously provider and consumer on a single WebSocket. Events never collide:

Provider Mode — receives user orders, delivers strategies
Consumer Mode — hires sub-agents, pays USDC escrow
Shared Outbound Set — tracks every negotiationId Maestro originates as consumer.
Race-Safe Event Bus — TTL buffer + waitFor(predicate) ensures zero lost events.
Fault-Tolerant — partial sub-agent failures still yield a complete strategy.
